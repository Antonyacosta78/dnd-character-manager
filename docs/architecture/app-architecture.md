# Architecture: App Structure

## Purpose

Capture the shared technical direction for the Next.js app.

## Current Plan

- Use Next.js App Router for the UI shell and routes.
- Keep domain logic outside UI components where possible.
- Separate app code, shared UI, and parsing/import code clearly.

## Notes

- Favor simple local architecture until persistence or backend needs force more structure.
- Keep import tooling isolated from runtime app code.
